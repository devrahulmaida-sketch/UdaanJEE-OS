# Multi-Tenant Open Deployment Model

## Tenant types
- Public cloud default (global community)
- School-hosted tenant
- NGO-hosted regional tenant
- Offline district edge node

## Isolation model
- Tenant-aware identity and scoped RBAC
- Data partition keys and row-level security
- Isolated object storage prefixes per tenant

## Federation model
- Opt-in anonymized aggregate analytics only
- Curriculum pack sharing through signed registries
- Tenant-level plugin approval workflows
