# Contributing to UdaanJEE OS

## Development workflow

1. Fork the repository and create a feature branch.
2. Link issues in commit messages.
3. Add tests for behavior changes.
4. Run lint and tests before opening a pull request.

## Commit style

Use Conventional Commits.

- `feat:` new feature
- `fix:` bug fix
- `refactor:` structural change without behavior change
- `docs:` documentation updates
- `test:` tests and fixtures
- `chore:` maintenance

## Code review checklist

- Accessibility compliance verified
- No hardcoded secrets
- Backward compatibility considered
- Observability metrics included for new services
- Error paths and edge cases tested

## Community guidelines

Follow `CODE_OF_CONDUCT.md` for all interactions.
